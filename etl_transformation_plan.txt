Transformation Stage Plan for the Student Productivity Dataset produced via copilot
============================================================

As a beginner data analyst, I would keep the Transformation stage simple and focus on the columns needed for the hypotheses.

1. Create a working copy of the raw data
- Start with a copy of the raw dataset so the original data remains unchanged.

2. Clean the Gender column
- Since Hypothesis 1 compares male and female students, I would remove missing values and exclude any "Other" values.

3. Handle missing values in numeric columns
- For columns such as Sleep_Hours_per_night, I would fill missing values with the column average.

4. Clean other numeric columns
- I would also fill missing values in columns such as:
  - Screen_Time
  - Social_Media_Usage
  - Physical_Activity_Hours
  - Prior_GPA
  - Productivity_Score

5. Standardise column names
- I would make column names consistent and easier to work with.

6. Check for duplicates
- I would remove any duplicate rows.

7. Create simple derived features if needed
- For example, I could create simple flags such as high screen time or high activity if these would help with the analysis.

8. Save the cleaned dataset
- Finally, I would save the transformed dataset for the analysis stage.

Why this approach is suitable for a beginner
-------------------------------------------
- It focuses on the columns used in the hypotheses.
- It keeps the cleaning process simple.
- It avoids overcomplicating the ETL workflow.
- It prepares the data for visualisation and analysis.

In simple terms, I would mainly:
- remove irrelevant gender values,
- fill missing numeric values with averages,
- remove duplicates,
- and save a clean dataset for analysis.
